1. 安装git

win7可以安装git的，并且git带了ssh、bash，注意git\usr\bin这个路径里才有ssh，配置环境变量
D:\SoftwareandTools\git\usr\bin;D:\SoftwareandTools\git\bin;（扯远了）

配置:
git config --global user.name "xxx"
git config --global user.email "xxx"

生成公钥：
ssh-keygen -t rsa -C "xx@xx"
进入 C:\Users\主机名\.ssh 目录，把 id_rsa.pub 里面的信息复制到GitHub的SSH公钥中即可

2. 安装Node.js+Hexo

介绍：

https://nodejs.org/en/download/releases

安装node

Hexo 是基于Node.js 的博客框架，就像 Java 要依赖 JDK 环境一样，所以要安装 Hexo 需要先安装 Node.js 。

win7电脑的问题：

  • 高版本（node-v18.17.1-win-x64）会有问题：
    ERROR A system error occurred: uv_os_gethostname returned ENOSYS (function not implemented)
    https://blog.csdn.net/fang_li_na/article/details/127537808
    uv_os_gethostname 不支持win7，解决办法：安装旧版本node或者换系统吧
    一些吐槽：
    https://www.cnblogs.com/c2soft/articles/17287664.html
    https://github.com/vuejs/core/discussions/6062
    https://github.com/libuv/libuv/issues/3260
    https://github.com/nodejs/node/issues/40353
  • 低版本（node-v12.22.12-win-x64）可以（同时Node.js高版本不支持win7），但要小心选择好各个重要包的版本。
  • 最终我的版本：
    E:\Desktop>node -v
    v12.22.12
    E:\Desktop>npm -v
    6.14.16
    E:\Desktop>hexo -v
    hexo-cli: 4.3.1

配置node环境变量

配置环境变量：
注意名字叫NODE_PATH，值是：D:\SoftwareandTools\nodejs\node-v18.17.1-win-x64;
然后加到系统环境变量的path里
这样做意义有两个，一是能够命令行输入node，二是代表库安装位置设置成了这个位置下的node_modules（我这样的好处是之后安装cnpm等不需要再配置环境变量好像，所以没有按照网上设置node_global文件夹）

这是命令行配置方式，也可以直接写文件"C:\Users\Administrator\.npmrc"
npm config set prefix "D:\SoftwareandTools\nodejs\node-v18.17.1-win-x64"
npm config set cache "D:\SoftwareandTools\nodejs\node-v18.17.1-win-x64\node_cache"  (需要新建文件夹node_cache)

最终npm root -g 显示 D:\SoftwareandTools\nodejs\node-v18.17.1-win-x64\node_modules

检查：
node -v
npm -v 这里如果报错就需要在"C:\Users\Administrator\.npmrc"里重新设置：
prefix=D:\SoftwareandTools\nodejs\node-v12.22.12-win-x64
cache=D:\SoftwareandTools\nodejs\node-v12.22.12-win-x64\node_cache
最终npm ls -g --depth=0会显示安装的包:
D:\SoftwareandTools\nodejs\node-v12.22.12-win-x64
+-- cnpm@7.1.1
+-- hexo-cli@4.3.1
`-- npm@6.14.16

安装 cnpm

你懂的，比npm方便(注意我是低版本node，要安装旧版本的cnpm！https://zhuanlan.zhihu.com/p/120159632)

不指定版本会默认安装最新的，然后报错：
npm install -g cnpm --registry=https://registry.npm.taobao.org
Unsupported engine for cnpm@9.2.0: wanted: {"node":">= 14.18.0"} (current: {"node":"12.22.12","npm":"6.14.16"})
我选择这个：
https://www.npmjs.com/package/cnpm/v/7.1.1
npm install cnpm@7.1.1 -g --registry=https://registry.npmmirror.com

安装Hexo

cnpm install -g hexo-cli

3. Hexo使用

hexo -v
hexo init blog
cd blog
npm install （如果必要的话，一般不用）
hexo server （简写 hexo s）

访问 http://localhost:4000/ 即可看到

项目主要文件目录介绍：
.
├── .deploy       # 需要部署的文件
├── node_modules  # 项目所有的依赖包和插件
├── public        # 生成的静态网页文件
├── scaffolds     # 文章模板
├── source        # 博客正文和其他源文件等都应该放在这里
|   ├── _drafts   # 草稿
|   └── _posts    # 文章
├── themes        # 主题
├── _config.yml   # 全局配置文件
└── package.json  # 项目依赖信息

新建文章，也可以直接在_posts文件夹下创建md文件
hexo new "第一篇Hexo blog文章"
Hexo框架文章中的图片只支持外链的形式，有很多方法解决，比如，可以在码云或者七牛云部署一个自己的图床。？

4. 关联github和gitee

在gitee新建一个仓库，仓库名和路径必须是用户名！
在github新建一个仓库。仓库名一样必须是用户名！
添加自己电脑的公钥！

在项目根目录下安装git部署插件 之后就可以使用 hexo deploy（或简写 hexo d）将项目部署到远程仓库
win7安装之前在 https://www.npmjs.com/package/hexo-deployer-git?activeTab=readme 查查安装哪个旧版本！

cnpm install hexo-deployer-git@3.0.0 --save

修改项目配置文件：_config.yml，在最下面，修改如下内容

# Deployment  https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repo: git@github.com:summer-I-love-NKU/summer-I-love-NKU.github.io.git
  branch: master

使用命令 hexo d 将项目部署到 gitee 远程仓库，此时本地文件夹中出现有一个public文件夹。

github直接就能访问自己的网站了 https://summer-i-love-nku.github.io/

gitee需要开启 Gitee Pages 静态网页托管服务（要先身份证、手持身份证的认证审核），每次重新上传代码到gitee时，都需要点击更新按钮重启page服务，比较麻烦。

5. Hexo修改主题

官网主题：https://hexo.io/themes/
可以去官网找自己喜欢的主题，下载下来，将里面的文件复制到自己项目的 theme 目录下

修改根目录下的配置文件 _config.yml里：
theme: xxx

启动项目，访问：http://localhost:4000/ 先在本地查看主题是否修改成功

目前使用了 https://autoload.github.io/about/
哦不我换成了这个！！！ https://blinkfox.github.io/

# 将修改后的项目部署到远程仓库
# 生成静态网站文件
hexo g  
# 上传到远程仓库
hexo d  
# 1、清除 hexo 的缓存
hexo clean
# 2、采用一键部署
hexo g --d

_config.yml一些其他配置：

# 如何上传README文件
# _config.yml里针对特定文件让解释器跳过渲染（hexo g -d 命令会使.md 文件会被转化成 HTML 文件）
skip_render: README.md

# url
url: https://summer-i-love-nku.github.io

至此，Hexo + GitHub 部署自己的个人博客完成！

附录：学习node和npm

学习一些命令：
https://docs.npmjs.com/cli/v6/commands
查看需要安装哪个版本：
https://www.npmjs.com/package/

npm install xxx：
安装项目到项目目录下，不会将模块依赖写入devDependencies或dependencies。
npm install -g xxx:
-g的意思是将模块安装到全局，不是安装到当前目录的项目下
npm install -save xxx:
--save 等同于-S （常用，可保存在package.json文件中），-S, --save 安装包信息将加入到dependencies（生产阶段的依赖,也就是项目运行时的依赖，就是程序上线后仍然需要依赖）
npm install -save-dev xxx:
--save-dev 等同于 -D，-D, --save-dev 安装包信息将加入到devDependencies（开发阶段的依赖，就是我们在开发过程中需要的依赖，只在开发阶段起作用。）


显示安装的包
npm list -g
npm ls --depth=0 -g

注意安装和卸载，
npm uninstall -g xxx

查看服务器可安装的包的版本
npm view cnpm versions

Hexo+Gitlab搭建步骤

参考：
主要教程 https://blog.csdn.net/hubojing/article/details/125237882
示例，但没什么帮助 https://gitlab.com/pages/hexo
runner https://docs.gitlab.com/runner/register/index.html
一个奇怪问题解决 https://stackoverflow.com/questions/53370840/this-job-is-stuck-because-the-project-doesnt-have-any-runners-online-assigned

注册gitlab似乎要FQ才能顺利注册，不然可能要验证手机号，但页面里没有中国的手机号。
因为我用github注册有问题，我直接用了新的邮箱注册。

步骤

按照教程 ，先创建仓库：用户名.gitlab.io，记得在settings-Repository-Protected branches打开允许强制推送，方便之后操作！

  1. 先在CI/CD Settings 关闭共享的runner，创建一个runner，才能获得token

  2. 然后按照指示 https://docs.gitlab.com/runner/install/windows.html 下载exe文件 （Download the binary for 64-bit）

  3. 然后新建一个gitlab-runner文件夹存放exe文件(最好不要放项目里)，在命令行执行这个：
    E:\Desktop>gitlab-runner-windows-amd64 register
    输入url（https://gitlab.com）和token（刚刚获取的），name随便起，executor选择shell

  4. 最后执行 gitlab-runner-windows-amd64 run
    看到Project runners绿色就ok了
    （可以点进去设置一下，勾选这两项，防止出现这个问题）
    Active Paused runners don’t accept new jobs
    Run untagged jobs Indicates whether this runner can pick jobs without tags

  5. 在项目中增加.gitlab-ci.yml文件！

    image: node:12.22.12
cache:
  paths:
    - node_modules/

before_script:
  - npm install hexo-cli -g
  - npm install
  - npm uninstall hexo-prism-plugin
  - npm install hexo-generator-search --save
  - npm i hexo-permalink-pinyin --save
  - npm i --save hexo-wordcount
  - hexo generate

pages:
  script:
    - hexo generate
  artifacts:
    paths:
      - public
  rules:
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH

  6. 之后把源文件上传到gitlab

这些文件
_config.yml  node_modules(不需要上传，本地调试使用，加入gitignore)  package-lock.json  source  
db.json      package.json  scaffolds          themes    .gitignore  .git


注意.gitignore不要写这仨文件：
db.json
package.json
package-lock.json

一些问题

https://blog.csdn.net/gongdamrgao/article/details/115032436
关于git的问题：error: src refspec main does not match any
执行git branch -m master main，可以看到master已经被成功改为main了，如此，我们继续push就不会有什么问题了
ERROR: Job failed (system failure): prepare environment: failed to start process: exec: “pwsh”: executable file not found in %PATH%.

打开config.toml文件，
shell = "pwsh"
改为：
shell = "powershell"
